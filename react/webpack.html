Webpack

Webpack은 JavaScript 애플리케이션을 위한 모듈 번들러(module bundler)입니다. 모듈 번들러란 애플리케이션을 구성하는 다양한 모듈들을 하나로 묶어주는 도구입니다. 웹 개발에서 자바스크립트 파일뿐만 아니라 CSS, 이미지, 폰트 등 다양한 파일들도 모듈로 취급하여 번들링할 수 있습니다.

Webpack은 모듈 간의 의존성 관리를 자동으로 해주고, 필요한 모듈들을 로드하고 변환하여 최종 번들 파일을 생성합니다. 이렇게 번들링된 파일은 웹 애플리케이션에서 사용되며, 애플리케이션의 성능을 향상시키고 파일 간 충돌을 방지하는 데 도움이 됩니다.

Webpack은 다양한 기능과 확장성을 제공합니다. 주요 기능으로는 코드 스플리팅(code splitting), 모듈 로딩, 플러그인 시스템, 개발 서버, 자동 리로딩 등이 있습니다. 이러한 기능들을 활용하여 개발자는 웹 애플리케이션을 구성하고 관리할 수 있습니다.

Webpack은 주로 Node.js 기반의 환경에서 사용되며, 대부분의 프론트엔드 프레임워크 및 도구들에서도 널리 채택되고 있습니다. 예를 들면 React, Angular, Vue.js 등의 프레임워크에서도 Webpack을 사용하여 애플리케이션을 구성합니다.

Webpack은 강력한 기능과 유연성을 제공하여 복잡한 프론트엔드 프로젝트의 모듈 관리와 번들링을 용이하게 해주는 도구입니다.

1.4 Webpack.config.js(기본값) 설정

npx webpack을 터미널에 작성하면 파일이 번들링(만들) 됩니다.

webpack.config.js 파일 만들기
const path = require('path');

mode: 'development',  production//소스압축해서보기,  development//개발소스,
module.exports = {
    entry: {
        index: './src/index.js'   //소스를 넣을 곳
    },
    output: {
        path: path.resolve(__dirname, 'public'),
        filename: 'js/[name].js'  //소스를 패치해서 나오는곳
    }
}
만든후 터미널에서 npx webpack 실행하면 파일을 만들어 줍니다.

    const HtmlWebpackPlugin = require('html-webpack-plugin');

    module.exports = {
        mode: 'production',
        entry: {
            index: './src/index.js'
        },
        output: {
            path: path.resolve(__dirname, 'public'),

                    exclude: '/node_modules'
                },
            ],
        },
